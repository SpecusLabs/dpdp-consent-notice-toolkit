## Context: Why AI Products Have Special Disclosure Obligations

When a user signs up for a standard SaaS product, they expect their data to be used to provide the service. When a user signs up for an AI product, the data use is often more complex and less obvious:

- User inputs may be sent to a third-party AI API (OpenAI, Anthropic, Google Vertex, etc.)
- Inputs may be used to improve or fine-tune a model
- Inference logs may be retained by the AI provider
- The product may analyse patterns across all users to improve its own algorithms
- Outputs generated by AI may be based on other users' data

Under DPDP, users must be told all of this in plain language, before they consent. Vague phrases like "we use AI to improve your experience" are not sufficient.

---

## Key Disclosure Requirements for AI Products

1. **Which AI APIs you use** — Name the provider (OpenAI, Anthropic, Google, etc.)
2. **What data is sent to them** — Exactly what user data reaches that API
3. **Whether it is used for training** — Many APIs have a setting to opt out of training use; you must tell users whether you have opted out or in
4. **Inference logs** — Whether the AI provider retains logs of API calls and for how long
5. **Output disclaimer** — AI outputs may be inaccurate; users should not rely on them for professional or legal decisions
6. **Separate consent for training** — If you use user data to train your own models, this requires separate, explicit consent

---

## Where to Place This Notice

- **At signup** — Before the user creates an account
- **Before first use of an AI feature** — If a user accesses an AI feature for the first time, give a contextual notice before they submit their first input
- **In your onboarding flow** — Within the first session, before the user provides any significant data
